# 가격 변경 후 가입 전환율 개선 여부 분석

## 결론: 전체적으로는 개선처럼 보이지만, 실제로는 두 세그먼트 모두 악화됨 (Simpson's Paradox)

**정의**: 전환율 = signups / visitors, 합산(sum-of-signups / sum-of-visitors) 기준. 기준일 2026-03-01 (포함) 이후를 "post", 이전을 "pre"로 분류.

## 전체 지표만 보면
- Pre (2026-01-05~02-28, n=107건): 63,510 visitors / 3,224 signups = **5.08%**
- Post (2026-03-01~04-25, n=109건): 70,002 visitors / 4,081 signups = **5.83%**
- 절대 +0.75%p, 상대 **+14.8% 상승** → 언뜻 성공으로 보임

## 그러나 세그먼트별로 쪼개보면 정반대
| 세그먼트 | Pre 전환율 (n) | Post 전환율 (n) | 변화 |
|---|---|---|---|
| Enterprise | 3.02% (53) | 2.60% (54) | **-13.9%** |
| SMB | 7.96% (54) | 7.17% (55) | **-9.9%** |

두 세그먼트 모두 전환율이 하락했다. 표본 수(각 53~55건)도 노이즈로 치부하기엔 충분히 크다.

## 원인: 세그먼트 믹스 이동 (Simpson's Paradox)
SMB는 원래 전환율이 훨씬 높음(~7-8% vs Enterprise ~2.5-3%). Post 기간에 방문자 구성비가 크게 바뀌었다:
- Pre: Enterprise 58.4% / SMB 41.6%
- Post: Enterprise 29.4% / SMB 70.6%

즉 전환율이 낮은 Enterprise 비중이 줄고 전환율이 높은 SMB 비중이 늘어난 결과, 실제로는 두 그룹 다 나빠졌는데도 전체 평균은 좋아 보이는 착시가 발생했다. 가격 변경과 방문자 구성비 변화가 같은 시점에 겹쳤을 가능성이 있어(마케팅 채널 변화 등 confounder 후보), 이 데이터만으로는 가격 변경이 구성비 변화를 유발했는지, 아니면 구성비가 우연히 같이 바뀐 것인지 인과관계를 판단할 수 없다.

## 데이터 품질 이슈 및 처리
- **완전 중복 행 8쌍(16행)**: date+segment+visitors+signups 모두 동일한 행이 8건 발견. 중복 제거(첫 행만 유지) → 230행 → 222행.
- **signups 결측 6건**: visitors만 있고 signups가 NaN인 행 6건 발견(pre 2건, post 4건). 전환율 계산 불가하므로 해당 행 제외 → 222행 → 216행 (pre 107 / post 109, 균형 유지됨).
- 그 외 signups > visitors, 음수값 등 이상치는 없음.

## 권고
1. 현재 데이터 기준으로 **가격 변경이 전환율을 개선했다고 결론 내릴 수 없다** — 오히려 두 세그먼트 모두 하락했다.
2. 전체 평균이 아닌 **세그먼트별 전환율을 핵심 KPI로 추적**할 것. 믹스 변화가 결론을 왜곡할 수 있다.
3. Post 기간에 SMB 비중이 급증한 원인(마케팅/트래픽 소스 변화 등)을 별도로 조사해 confounder 여부를 확인 필요.
