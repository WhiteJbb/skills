# Research Boost

Blocks the 4 causes of shallow analysis: single-angle framing, confirmation bias, ungraded confidence, correlation-as-cause. Depth = the strength of what you tried to break, not the length of what you wrote.

## Language
User-facing responses in Korean. Cited material stays in its source language with a Korean gloss where needed.

## 1. Question contract
- One line: the decision this analysis informs, and for whom. An analysis with no decision attached drifts into a book report.
- Decompose into 3-6 sub-questions; per sub-question, one line: what evidence WOULD answer it. Evidence-first framing prevents collecting whatever is easiest to find.

## 2. Evidence discipline
- Every claim gets a grade at capture time: OBSERVED (you saw/ran it) / SOURCED (citation at hand) / INFERRED (your reasoning, shown) / ASSUMED (declared). Never upgrade a grade silently.
- Quotes and numbers re-verified against the source before use — never from memory of the first read.
- Provenance per fact: where it came from. A fact you cannot source is INFERRED, not SOURCED.

## 3. Adversarial pass (the core gate — do not skip)
- For the emerging conclusion, search for DISconfirming evidence with the same effort spent confirming. Write the strongest counter-case (steelman), then answer it — or weaken the conclusion.
- Causal claims: list 2+ rival explanations (confounder, reverse causation, selection, coincidence) and what rules each out — or state that it can't be ruled out. Cause stated without this pass is a rule violation, not a style choice.
- Wide question AND high stakes: spawn 2-3 parallel subagents with forced-diverse priors (skeptic / advocate / base-rates), each returning a ≤10-line case; reconcile explicitly. Otherwise the written counter-case suffices.

## 4. Synthesize
- Findings ranked by confidence, grade visible per claim. Unknowns and open questions are first-class output — state what evidence would change the conclusion.
- The conclusion answers the step-1 decision directly. Hedging on everything equals concluding nothing — commit where the evidence supports it.

## 5. Report
- Deliverable hygiene: if the analysis is delivered as a document/file, it contains only the analysis — no status line, no metadata footer inside it. The status line goes in your chat response.
- End the chat response with: `sub-questions: K/N answered | claims: O/S/I/A counts | counter-case: written & answered | rivals ruled out: R/T`

## Never
Conclude from one angle · cite from memory · upgrade an inference to fact · state cause without rivals named · hide unknowns · equal-hedge everything to avoid committing
