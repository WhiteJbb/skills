# Report Boost

Blocks the 4 causes of weak reports: template-first structure, unsupported claims, buried conclusions, bullet-fragment prose. A report is an argument, not a container.

## Language
User-facing responses in Korean. The report follows the requested language.

## 1. Thesis contract (before any outline)
- Three lines: reader / the ONE sentence they must walk away believing (the thesis) / the decision or action it informs.
- Cannot write the thesis in one sentence → not ready to outline; gather or think more first.

## 2. Evidence ledger (before drafting)
- List every claim the report will make. Tag each: DATA (evidence at hand) / SOURCE (doc or citation) / ASSUMPTION (declared as such).
- An ASSUMPTION presented as fact is the #1 credibility bug. Assumptions get their own visible line in the report.
- One canonical value per figure — a number used twice must match everywhere.

## 3. Outline gate
- Each section: one line stating what it PROVES toward the thesis. A section that proves nothing (unrequested background, "overview", filler) gets cut — template completeness is not a goal.
- Order: conclusion first, evidence in decreasing importance, details/appendix last. A reader who stops after page 1 must leave with the thesis.
- The recommendation is an actionable ask, not a direction: state what the reader should approve/do and WHEN — when the notes contain real dates or timing constraints, stage the plan on them explicitly (judged gap vs reference: a memo that argued well but omitted the staged timeline and the ask).

## 4. Draft
- Paragraphs of complete sentences; each paragraph opens with its point. Bullets only for genuinely enumerable facts, never to carry an argument.
- Claims keep their tag's honesty: sourced claims name the source in place; inferred ones show the reasoning.
- Numbers, names, dates re-looked-up at write time, never from memory.
- No throat-clearing ("As is well known…", "In today's rapidly changing…") — delete on sight.

## 5. Done gate
- Thesis check: re-read each paragraph asking "does this advance the thesis?" Cut or move what doesn't.
- Figure sweep: every number cross-checked against the ledger; repeated figures identical.
- Naturalness pass: read the full text top-to-bottom as the READER; fix every sentence you stumble on.
- Nontrivial reports: ONE fresh-context subagent gets ONLY the thesis + final text (none of your reasoning): "does every section support this thesis? which claim lacks support?" Fix what it confirms; dismiss a false positive in one line.
- Deliverable hygiene: the report file contains ONLY what its reader should see — no status line, no process notes, no metadata footer inside the document (measured: a claims-count footer inside the memo lost a blind judging). The status line below goes in your chat response.
- End the chat response with: `claims: N (DATA d / SOURCE s / ASSUMPTION a) | sections cut: K | figures consistent: Y`

## Never
Outline before thesis · present an assumption as fact · bury the conclusion · bullet-fragment an argument · keep a section because templates have it · reuse a figure without re-checking
